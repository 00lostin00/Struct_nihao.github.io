排序和递归

Nov 23, 2024

冒泡，选择，插入，快排，归并，桶排， 先引入一个概念，时间复杂度 and csdn时间复杂度。 时间复杂度是衡量一个算法优秀的最重要的标准之一，也往往是我们在编写一个程序需要优化

时间复杂度定义：

衡量一个算法的快慢，一定要考虑数据规模的大小。所谓数据规模，一般指输入的数字个数、输入中给出的图的点数与边数等等。一般来说，数据规模越大，算法的用时就越长。而在算法竞赛中，我们衡量一个算法的效率时，最重要的不是看它在某个数据规模下的用时，而是看它的用时随数据规模而增长的趋势，即 时间复杂度。

用人话来说就是，程序中的语句执行次数，而在算法竞赛中，我们往往只关注循环和递归的时间复杂度。(后期有时间专门写个时间复杂度定义的轮子)

而时间复杂度的单位我们往往可以使用 $O$ 这个符合来表示，以上四个 $for$ 循环的时间复杂度为$O(10),O(n),O(n/2),O(n*m)$。我们可以把这里面的时间复杂度理解为一个 $f(x)$ 的函数，我们一般会只考虑最高次数项，其余项忽略不计（常数项也忽略不计）。

把以上四个 $for$ 循环当做一个程序，那个它是时间复杂度就是 $O(10+n+n/2+nm)$。忽略低次项，只考虑最高次项，即时间复杂度为 $O(nm)$ 。

同样我们往往也会把最高次项的系数忽略掉，如 $O(n/2)$ 我们往往认为时间复杂度为 $O(n)$ 。

一个算法的正确复杂度应该是最坏时间的复杂度。比如从 1-1000 中猜一个数字，最优复杂度可能是一次就猜中，最坏可能是猜 1000 次才猜对。那个这个算法的时间复杂度就是 $O(1000)$ 。

排序

排序算法（英语：Sorting algorithm）是一种将一组特定的数据按某种顺序进行排列的算法。排序算法多种多样，性质也大多不同。

我们往往是按照从大到小或者从小到大的排序，但是排序的准确定义是按照某种特定的顺序进行排序的，即你可以自己设定一个函数 $f(x)$ 进行排序，比如 $f(x) = |x-100|$ ，然后对 $f(x)$ 从小到大排序，那么这样的排序就显而易见就是离 100 越近越靠前。

接下来我们讲的是简单的对于权值的排序，即从大到小或者从小到大排序。

而排序的算法各种各样，有官方认可的，也有民间自创的，我们这里只讲一些常用的，同时也是在目前知识范畴下可以学到的。

选择排序

选择排序（英语：Selection sort）是一种简单直观的排序算法。它的工作原理是每次找出第 $i$ 小的元素（也就是 $A_{i..n}$ 中最小的元素），然后将这个元素与数组第 $i$ 个位置上的元素交换。

过程：就是每次挑出来一个最小的数放到前面

for(int i=1;i<=n;i++)
{
	int min_v=i;
	for(int j=i;j<=n;j++)
	{
		if(a[j]<a[i])min_v=j;
	}
	swap(a[i],a[min_v]);
}

时间复杂度 $O(n^2)$。

冒泡排序

冒泡排序（英语：Bubble sort）是一种简单的排序算法。由于在算法的执行过程中，较小的元素像是气泡般慢慢「浮」到数列的顶端，故叫做冒泡排序。

过程：

它的工作原理是每次检查相邻两个元素，如果前面的元素与后面的元素满足给定的排序条件，就将相邻两个元素交换。当没有相邻的元素需要交换时，排序就完成了。

经过 i 次扫描后，数列的末尾 i 项必然是最大的 i 项，因此冒泡排序最多需要扫描 i-1 遍数组就能完成排序，而单次扫描时间为 $O(n)$，总时间复杂度仍为 $O(n^2)$。

插入排序

插入排序（英语：Insertion sort）是一种简单直观的排序算法。它的工作原理为将待排列元素划分为「已排序」和「未排序」两部分，每次从「未排序的」元素中选择一个插入到「已排序的」元素中的正确位置。

	a[key]=a[i];
}

时间复杂度 $O(n^2)$。

计数排序

计数排序是一种特殊的桶排序。 计数排序的工作原理是使用一个额外的数组 C，其中第 i 个元素是待排序数组 A 中值等于 i 的元素的个数，然后根据数组 C 来将 A 中的元素排到正确的位置。 它的工作过程分为三个步骤：

  1. 计算每个数出现了几次；
  2. 求出每个数出现次数的前缀和；
  3. 利用出现次数的前缀和，从右至左计算每个数的排名。 这样时间复杂度是 $O(max(n,w))$，w 表示排序中最大的那个数。 计数排序的时间复杂度小，但空间复杂度极大，当 w 是大于 $10^7$ 时就不建议使用计数排序，会导致空间溢出。

递归

定义：

递归（英语：Recursion），在数学和计算机科学中是指在函数的定义中使用函数自身的方法，在计算机科学中还额外指一种通过重复将问题分解为同类的子问题而解决问题的方法。

引入：

递归的基本思想是某个函数直接或者间接地调用自身，这样原问题的求解就转换为了许多性质相同但是规模更小的子问题。求解时只需要关注如何把原问题划分成符合条件的子问题，而不需要过分关注这个子问题是如何被解决的。

递归代码最重要的两个特征：结束条件和自我调用。自我调用是在解决子问题，而结束条件定义了最简子问题的答案。

一个简单例子，求一个数的阶乘，往往我们会使用循环来求解，同样也可以使用递归来求解。 递归做法，我们求 n 的阶乘，就是求 n $\times$ fc(n-1)，求 n-1 的阶乘，就是求 n-1。

#include <bits/stdc++.h>
using namespace std;

int n;
int fc(int x)
{
    if (x == 1)//终止条件
        return 1;//最小子问题解
    return x * fc(x - 1);//缩小规模
}
int main()
{
    cin >> n;
    int res = 1;
    for (int i = 1; i <= n; i++)
        res *= i;
    cout << res << endl;//循环求得
    cout << fc(n) << endl;//递归求得
    return 0;
}

对于这个程序递归和循环的时间复杂度均为 $O(n)$。

        b = c;

//递归做法
int fb(int x)
{
	//x为1和2都是终止条件，理论上可以设置任意多个终止条件,但是没必要
	if(x==1)	return 1;
	
	if(x==2)    return 1;

	//if(x==3) return 2;
	return fb(x-1)+fb(x-2);//缩小范围
}

显然循环的时间复杂度是 $O(n)$。

那么这个递归求阶乘的时间复杂度为多少？ 这个是 5 的斐波那契数列的递归示意图，每走一步是一次，大概算一下，由于从 $n$ 到 $3$ 每一个数都被拆成两个数再递归求得，当 $n$ 比较大时，总复杂度是满足 $O(2^n)$ 。 F(5) ├── F(4) │ ├── F(3) │ │ ├── F(2) │ │ │ ├── F(1) │ │ │ └── F(0) │ │ └── F(1) │ └── F(2) │ ├── F(1) │ └── F(0) └── F(3) ├── F(2) │ ├── F(1) │ └── F(0) └── F(1)

显然这样递归效率要远远不如循环或者别的做法，所以写递归的时候要知道它的缺点：在程序执行中，递归是利用堆栈来实现的。每当进入一个函数调用，栈就会增加一层栈帧，每次函数返回，栈就会减少一层栈帧。而栈不是无限大的，当递归层数过多时，就会造成 栈溢出 的后果。同时当一次调用多个栈的时候，时间复杂度往往是指数级别的，会非常慢。

当然递归的优点就是递归允许将复杂问题分解为更小的子问题，这有助于简化问题解决过程（快排和归并就是用递归实现的），在后续学到的搜索（DFS）就是建立在递归的基础上，以及后面数据结构图和树遍历等。

只能说，递归非常重要，是检测一个人是否学懂编程的门槛。

快速排序

定义：

快速排序（英语：Quicksort），又称分区交换排序（英语：partition-exchange sort），简称「快排」，是一种被广泛运用的排序算法。

过程：

快速排序的工作原理是通过分治的方式来将一个数组排序。 对于一个数列，我们找到一个中间位置的值 $x$ ，通过划分区间，比 $x$ 小的放在左边，比 $x$ 大的放在右边。然后在分别递归这两个区间，按照上述做法，直到递归到区间只有 $1$ 个数时，终止递归，再由开始分出来的区间，依次合并，构成的序列就是一个完美的拍好序的数列，完结撒花。

void k_sort(int x, int y)
{
    int mid = a[(x + y) >> 1];
    int i = x, j = y;
    while (i <= j)
    {
        while (a[i] < mid)
            i++;
        while (a[j] > mid)
            j--;
        if (i <= j)
        {
            q = a[i], a[i] = a[j], a[j] = q, i++, j--;
        }
    }
    if (x < j)
        k_sort(x, j);
    if (i < y)
        k_sort(i, y);
    return;
}

时间复杂度分析： 在平均情况下，快速排序的时间复杂度是 $O(n \log n)$ 。这是因为每次划分都将数组分成两个大致相等的部分，递归的深度是 $log\ n$（其中 $n$ 是数组的长度），而每一层的划分操作需要 $O(n)$ 的时间。

快排在极端最坏情况下会退化成 $n^2$，当然这种情况很少，如果用到快排做题最好直接使用 sort 函数。

这边建议如果快排不太懂可以直接使用 sort 函数，很好用，后面的自定义排序也是基于这个函数当然比赛你也可以手搓快排。

归并排序

定义：

归并排序（merge sort）是高效的基于比较的稳定排序算法。 归并排序基于分治思想将数组分段排序后合并，时间复杂度在最优、最坏与平均情况下均为 $O(n\log n)$ 。

过程：

它和快排不同的是，快排是先处理区间，然后递归再合并。 归并是先递归区间，处理区间，合并区间。

归并排序最核心的部分是合并（merge）过程：将两个有序的数组 a[i] 和 b[j] 合并为一个有序数组 c[k]。 从左往右枚举 a[i] 和 b[j]，找出最小的值并放入数组 tmp[k]；重复上述过程直到 a[i] 和 b[j] 有一个为空时，将另一个数组剩下的元素放入 tmp[k]。 因为保证数组 a 和 b 均有序，所以处理区间合并的时间复杂度为 $O(n)$。

归并先一直把区间折半分开，直到区间中只有一个数，为终止条件，递归合并区间。

最后在把 tmp 中的数重新赋值给 数组 a。

void merge_sort(int l, int r)
{
    if (l == r)
        return;
    int i = l, mid = (l + r) / 2;
    int j = mid + 1;
    int k = i;
    merge_sort(l, mid);
    merge_sort(mid + 1, r);
    while (i <= mid && j <= r)
    {
        if (a[i] <= a[j])
            tmp[k++] = a[i++];
        else
            tmp[k++] = a[j++];
    }
    while (i <= mid)
        tmp[k++] = a[i++];
    while (j <= r)
        tmp[k++] = a[j++];
    for (int h = l; h <= r; h++)
        a[h] = tmp[h];
}
